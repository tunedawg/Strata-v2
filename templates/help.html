How to use Strata

Index workflow

1
Create a production. Enter a clear dataset name and click the add button. Use names that match the production, matter, custodian, or review batch.
2
Add documents. Use Add Files for selected files or Add Folder for a complete production folder. Strata keeps the files local.
3
Build the index. Strata extracts searchable text, uses OCR fallback for image PDFs/images where available, and persists the index for later use.
4
Search and review hits. Add one term per row. Each row runs independently and results are grouped by term.
5
Open and export. Click page/chunk chips to preview the source document at that page. Export XLSX or PDF hit reports for the current search set.

Local storage

Datasets, indexes, exports, logs, and metadata stay on this device under Documents/Strata unless configured otherwise.

Tip: Rebuild the index after adding or removing source files so result counts and page links stay accurate.

Production workflow

1
Select an indexed dataset. Production search uses the index built on the Index tab.
2
Enter search terms. Use witness names, entities, email addresses, dates, phrases, or Boolean strings. Each row is grouped separately.
3
Review term groups. Expand a group to see matching documents and clickable page/chunk chips.
4
Select responsive documents. Use individual checkboxes or Select all per group. A document can be responsive to multiple terms.
5
Export ZIP. The package includes selected documents and a responsiveness log with term, document, page/chunk, timestamp, and dataset metadata.

Review use

Production is best for preparing witness packets, issue sets, deposition exhibits, or responsive document pulls from an indexed dataset.

Redaction workflow

1
Select a dataset. Redaction scans read uploaded PDF documents directly; an index is not required.
2
Choose detection scope. Scan for Adobe PDF /Redact annotations, manual black PDF redaction rectangles, and drawn or sharpie-style marks on scanned pages.
3
Run the scan. Progress and the current file are shown in the sidebar. Cancel remains available during long scans.
4
Review findings. Results are grouped by document with clickable page links, finding type, location, and detail columns.
5
Export results. Export redacted documents as ZIP, a detailed findings list, an XLSX summary report, or a PDF report.
Tip: Drawn/sharpie detection is visual and heuristic. Treat it as a review aid and confirm final calls manually.

How search rows work

Each search row runs independently. Results are grouped under the exact row text, so use separate rows when you want separate counts, exports, or responsiveness groups.

Plain words

contract finds documents containing contract. Search is case-insensitive, so Contract and contract are treated the same.

Exact phrases

"wrongful termination" finds those words together in that order. Use quotes for names, clauses, subjects, or phrases that should not be split apart.

"John Smith" and "Project Falcon" are common examples.

AND

termination AND severance requires both terms in the same searchable chunk/page. Use AND to narrow results.

"John Smith" AND complaint finds chunks with both the exact name and complaint.

OR

fired OR terminated finds either word. Use OR for synonyms, spelling variants, related names, or alternate phrasing.

attorney OR counsel OR lawyer

NOT

contract NOT draft finds contract but excludes chunks containing draft. Use NOT carefully; it can remove useful hits if the excluded word appears nearby.

privilege NOT log

AND NOT

termination AND NOT severance requires termination and excludes severance. This is clearer than using NOT alone in longer searches.

"John Smith" AND NOT "John Smith Jr."

Wildcards

terminat* matches termination, terminated, terminate, and terminating. Wildcards are best for word stems.

investigat* matches investigate, investigation, investigated, and investigative.

Fuzzy search

~termination finds close misspellings or OCR variants. Use fuzzy search for scanned productions, names, and words often misread by OCR.

~McConnel may help find McConnell or OCR-damaged variants.

Proximity

"fired" W/10 "email" finds fired within 10 words of email. Use proximity when words should appear near each other but not necessarily as an exact phrase.

confidential W/5 agreement and "trade secret" W/20 disclosure

Useful combinations

("fired" OR terminated) AND "John Smith"

terminat* AND (email OR memo OR letter)

"performance improvement plan" OR PIP

~harassment AND NOT training

"confidential" W/10 "settlement"

Practical review tips

Use broad rows first to understand volume, then narrower rows with phrases, AND NOT, or proximity. Keep separate legal issues in separate rows so exports and Production groups remain easy to read.

Syntax reminder: Supported operators include AND, OR, NOT, AND NOT, exact phrases, trailing wildcards, fuzzy terms, and W/N proximity.
